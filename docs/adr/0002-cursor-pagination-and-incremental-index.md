# ADR 0002：游标分页与增量媒体索引

## 状态

已接受。

## 决策

Android 的 `/api/v1/media` 使用键集分页。排序键固定为
`(DATE_MODIFIED DESC, _ID DESC)`，游标是不透明、带版本前缀的令牌，记录上一页
最后一项的这两个值。后续页只返回排序键严格小于游标的项目，因此相同修改时间的
项目仍可稳定分页，不依赖易漂移的 offset。

媒体 ID 由媒体类型和 MediaStore ID 构成。新增项目会按排序键进入时间线；修改项目
可能移动到更靠前的位置；删除项目不会作为普通列表项返回。

Windows 在 SQLite 中保存 `devices`、`media_items`、`sync_cursors` 和
`schema_migrations`：

- 首次同步遍历所有页面，以 `(device_id, remote_id)` 幂等写入；
- 完成同步后保存第一项的 ID 与修改时间作为高水位检查点；
- 后续同步从第一页开始，遇到未变化的检查点即停止，因而无变化的启动只读取一页；
- 若远端数量减少，直接完整扫描；若增量写入后本地数量大于远端数量（例如一增一删），
  再执行完整扫描；
- 完整扫描为每项记录同一个 `last_seen_at`，完成后删除本轮未见的项目。

数据库更新和检查点提交均使用事务。同步中断时不会推进检查点；已写入的项目可在下次
同步时安全地再次 upsert。

## 结果

该方案无需手机端维护变更日志即可让常见启动路径保持增量，同时能通过数量变化和完整
对账处理删除。极端情况下，若远端在一次分页会话中持续变化，当前会话保持键集分页的
无重复语义，新变化由下一次同步获取。

SQLite 索引是展示和搜索的数据源，因此手机离线时仍可按文件名、相册或相对路径搜索
上一次成功索引的元数据。
