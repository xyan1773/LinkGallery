# ADR 0001：Android 媒体边界保持只读

- 状态：已接受
- 日期：2026-06-30

## 背景

LinkGallery 需要浏览和复制用户的私人媒体。删除、重命名、移动、编辑和上传会显著
扩大误操作、权限滥用和安全漏洞的影响面。

## 决策

Android Companion 仅提供设备信息、媒体元数据、缩略图和原文件字节流。复制任务、
去重、本地命名、临时文件和完成校验全部由 Windows 客户端负责。

`/api/v1` 不定义媒体写入端点。配对状态管理可以使用写请求，但不能修改相册内容。

## 结果

- Android 权限和测试范围更小。
- Windows 是所有复制状态的唯一事实来源。
- 未来若加入写功能，必须使用新的协议能力和单独 ADR，不能静默扩展现有接口。
